# 技術者向けドキュメント

技術者向けの正本はソースコードとテストです。この章は、ソースだけでは読み取りにくい設計意図、制約、検証手順への入口です。

## 位置づけ

- ソースコード: 実装の正本
- テスト: 実行可能な仕様
- ADR: 変更しにくい設計判断と不変条件
- Findings: 実装時の調査、判断、再現手順
- Spike: 採否判断のための実験記録

## 書くべきもの

技術者向け docs に書くのは、ソースを読んでも復元しにくい情報に限定します。

- なぜその設計にしたか
- 何を壊してはいけないか
- ビルド、配布、検証の手順
- 外部サービスやデータ配置の前提
- 事故時の切り分け手順

細かい関数仕様、内部 API、実装追従が必要な説明はソースとテストに寄せます。
