# 05 シナリオと銘柄

戦略コードは「どう売買するか」を書きます。「いつ・どの銘柄を・いくらの資金で検証するか」は
**シナリオ**として戦略とは別に設定します。

## シナリオの中身

| 項目 | 意味 |
|---|---|
| 銘柄（universe） | 検証対象の銘柄リスト（例 `7203.T`） |
| 開始日 / 終了日 | 検証期間（`YYYY-MM-DD`） |
| 足種（granularity） | `Daily` / `Minute` など |
| 初期資金 | 走り始めの現金 |

## 設定のしかた

通常は Strategy Editor の **Replay 設定**で入力します。

1. 戦略ファイルを開く。
2. 開始日・終了日・足種・初期資金を入力する。
3. Universe に検証したい銘柄を追加する。
4. Run。

!!! note "銘柄候補は終了日のカタログから"
    Universe の候補は、設定した**終了日時点**のローカルカタログから取得します。終了日が
    未設定だと候補を確定できません。

## どこに保存されるか

シナリオは戦略ファイルと同名の **サイドカー `.json`** に保存されます（戦略 `.py` の隣）。
通常は UI が書き出すので手で編集する必要はありませんが、中身はこの形です。

```json
{
  "scenario": {
    "schema_version": 3,
    "instruments": ["7203.T"],
    "start": "2024-01-01",
    "end": "2024-12-31",
    "granularity": "Daily",
    "initial_cash": 1000000
  }
}
```

戦略コードと検証条件を分けておくことで、**同じ戦略を別の期間・別の銘柄で**そのまま回せます。

## 確認

期間や銘柄を変えて Run し、対象データが切り替わることを確認します。
